# PPTTalK 测试规范

## 1. 测试类型

### 1.1 单元测试

#### 前端单元测试
- **框架**: Vitest
- **测试范围**:
  - 组件功能测试
  - 工具函数测试
  - 状态管理测试
  - 路由测试

#### 后端单元测试
- **框架**: pytest
- **测试范围**:
  - 核心功能模块测试
  - 工具函数测试
  - API 接口测试
  - 模型加载和推理测试

### 1.2 集成测试

#### 前端集成测试
- **框架**: Vitest + Testing Library
- **测试范围**:
  - 组件间交互测试
  - 页面导航测试
  - 表单提交测试
  - API 调用测试

#### 后端集成测试
- **框架**: pytest + FastAPI TestClient
- **测试范围**:
  - 模块间集成测试
  - API 与业务逻辑集成测试
  - 数据库操作测试

### 1.3 端到端测试

- **框架**: Playwright
- **测试范围**:
  - 完整用户流程测试
  - 跨浏览器兼容性测试
  - 响应式布局测试
  - 性能测试

## 2. 测试覆盖率目标

### 2.1 前端测试覆盖率
- **整体覆盖率**: ≥ 80%
- **关键功能覆盖率**: ≥ 90%
- **组件测试覆盖率**: ≥ 85%
- **工具函数覆盖率**: ≥ 95%

### 2.2 后端测试覆盖率
- **整体覆盖率**: ≥ 75%
- **核心功能覆盖率**: ≥ 90%
- **API 接口覆盖率**: ≥ 85%
- **工具函数覆盖率**: ≥ 90%

## 3. 测试用例设计原则

### 3.1 前端测试用例设计
- **边界值测试**: 测试输入的边界情况
- **异常测试**: 测试错误处理和异常情况
- **功能测试**: 测试核心功能的正确性
- **用户场景测试**: 测试真实用户使用场景
- **性能测试**: 测试组件渲染和交互性能

### 3.2 后端测试用例设计
- **输入验证测试**: 测试输入参数的验证
- **业务逻辑测试**: 测试核心业务逻辑的正确性
- **错误处理测试**: 测试错误处理机制
- **性能测试**: 测试 API 响应时间和资源使用
- **安全性测试**: 测试 API 的安全性

## 4. 测试执行机制

### 4.1 本地测试
- **前端测试**: `npm test`
- **后端测试**: `pytest`
- **端到端测试**: `npm run test:e2e`

### 4.2 CI/CD 测试
- **集成测试**: 每次提交时执行
- **端到端测试**: 每次合并到主分支时执行
- **覆盖率检查**: 每次提交时执行，低于目标值则失败

### 4.3 测试环境
- **开发环境**: 本地开发机器
- **测试环境**: CI/CD 环境
- **生产环境**: 生产服务器（仅执行监控测试）

## 5. 测试报告机制

### 5.1 测试报告格式
- **前端测试报告**: Vitest 生成的 HTML 报告
- **后端测试报告**: pytest 生成的 HTML 报告
- **端到端测试报告**: Playwright 生成的 HTML 报告
- **覆盖率报告**: Istanbul 生成的 HTML 报告

### 5.2 报告存储
- **本地报告**: 存储在 `coverage` 目录
- **CI/CD 报告**: 存储在 CI/CD 平台的 artifact 中
- **历史报告**: 存储在测试报告系统中，用于趋势分析

### 5.3 报告分析
- **覆盖率趋势分析**: 跟踪代码覆盖率的变化
- **测试执行时间分析**: 识别慢测试
- **失败测试分析**: 分析失败测试的原因
- **性能分析**: 分析测试执行性能

## 6. 测试工具配置

### 6.1 前端测试配置
- **Vitest 配置**: `vite.config.ts` 中的测试配置
- **ESLint 配置**: 确保测试代码符合代码风格

## 8. 测试最佳实践

### 8.1 前端测试最佳实践
- 使用测试替身（mock）模拟外部依赖
- 测试组件的行为，而不是实现细节
- 使用 Testing Library 的查询方法，模拟用户真实操作
- 保持测试的独立性和可重复性
- 编写清晰、可维护的测试代码

### 8.2 后端测试最佳实践
- 使用 pytest 的 fixtures 管理测试资源
- 测试 API 的响应，而不是内部实现
- 使用 mock 模拟外部服务和模型
- 保持测试的原子性和独立性
- 编写参数化测试，覆盖多种输入情况

### 8.3 端到端测试最佳实践
- 测试关键用户流程
- 使用页面对象模式组织测试代码
- 避免测试实现细节
- 保持测试的稳定性和可靠性
- 合理设置测试超时时间

本测试规范旨在确保 PPTTalK 项目的代码质量和功能可靠性。通过实施全面的测试策略，我们可以：
