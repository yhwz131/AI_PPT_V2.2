# PPTTalK 项目集成规范

## 1. 集成流程

### 1.1 新功能集成流程

1. **需求分析与规划**
   - 明确新功能的业务需求和技术要求
   - 确定新功能与现有系统的集成点
   - 评估集成风险和影响范围

2. **设计阶段**
   - 设计新功能的接口规范
   - 设计数据模型和流程
   - 确定与现有系统的交互方式

3. **开发阶段**
   - 实现新功能代码
   - 遵循项目的代码规范
   - 确保与现有系统的兼容性

4. **测试阶段**
   - 编写单元测试和集成测试
   - 进行端到端测试
   - 验证新功能与现有系统的集成效果

5. **部署阶段**
   - 更新部署脚本和配置
   - 部署新功能到测试环境
   - 验证部署效果

6. **上线阶段**
   - 部署新功能到生产环境
   - 监控系统运行状态
   - 收集用户反馈

### 1.2 服务集成流程

1. **服务注册**
   - 在 `start_all.sh` 中注册新服务
   - 配置服务的环境、端口、工作目录等信息

2. **服务启动**
   - 确保服务按正确的顺序启动
   - 实现服务健康检查机制

3. **服务通信**
   - 定义服务间的通信协议
   - 实现服务间的调用逻辑

4. **服务监控**
   - 集成到现有的监控系统
   - 实现服务状态的实时监控

## 2. 接口设计原则

### 2.1 RESTful API 设计原则

1. **资源导向**
   - 使用名词表示资源
   - 避免使用动词
   - 保持 URL 的简洁性

2. **HTTP 方法**
   - `GET`：获取资源
   - `POST`：创建资源
   - `PUT`：更新资源
   - `DELETE`：删除资源

3. **路径规范**
   - 使用小写字母
   - 单词间用连字符（-）分隔
   - 避免使用下划线和驼峰命名

4. **查询参数**
   - 使用 `Query` 装饰器
   - 保持参数名称的一致性

5. **请求体**
   - 使用 Pydantic 模型
   - 定义明确的数据结构

6. **响应格式**
   - 统一响应格式
   - 包含状态码、消息和数据

### 2.2 前端 API 调用规范

1. **API 客户端**
   - 使用 `apiGet`、`apiPost`、`apiUpload`、`apiDelete` 函数
   - 统一错误处理

2. **环境配置**
   - 通过 `.env` 文件配置 API 基础 URL
   - 支持不同环境的配置

3. **类型定义**
   - 使用 TypeScript 定义接口类型
   - 确保类型的一致性

4. **错误处理**
   - 统一错误处理逻辑
   - 提供友好的错误提示

## 3. 兼容性要求

### 3.1 版本兼容性

1. **API 版本控制**
   - 支持 API 版本的向后兼容
   - 提供版本迁移指南

2. **依赖兼容性**
   - 确保新功能的依赖与现有系统兼容
   - 避免依赖冲突

### 3.2 数据兼容性

1. **数据格式**
   - 保持数据格式的一致性
   - 支持数据的向后兼容

2. **数据迁移**
   - 提供数据迁移方案
   - 确保数据的完整性

### 3.3 服务兼容性

1. **服务接口**
   - 保持服务接口的稳定性
   - 避免破坏性变更

2. **服务依赖**
   - 明确服务间的依赖关系
   - 提供依赖版本的兼容性要求

## 4. 集成测试验证机制

### 4.1 测试策略

1. **单元测试**
   - 测试单个功能模块
   - 确保模块的正确性

2. **集成测试**
   - 测试模块间的交互
   - 确保集成的正确性

3. **端到端测试**
   - 测试完整的业务流程
   - 确保系统的整体功能

### 4.2 测试工具

1. **前端测试**
   - 使用 Vitest 进行单元测试
   - 使用 Testing Library 进行组件测试

2. **后端测试**
   - 使用 pytest 进行单元测试
   - 使用 FastAPI 的测试客户端进行 API 测试

3. **集成测试**
   - 使用端到端测试工具
   - 模拟真实的用户场景

### 4.3 测试流程

1. **测试环境**
   - 搭建独立的测试环境
   - 确保测试环境与生产环境一致

2. **测试执行**
   - 执行单元测试
   - 执行集成测试
   - 执行端到端测试

3. **测试报告**
   - 生成测试报告
   - 分析测试结果
   - 提供测试覆盖率

### 4.4 持续集成

1. **CI/CD 流程**
   - 集成到持续集成系统
   - 自动执行测试
   - 自动部署
