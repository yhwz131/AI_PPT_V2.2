# PPTTalK 竞赛优化建议与迭代路线

本文档整理针对**省级职业院校技能大赛一等奖**目标的优化建议，涵盖技术升级、产品完善、答辩策略和材料准备四个维度，供后续迭代参考。

---

## 一、项目定位与竞争力评估

### 1.1 核心亮点（已实现）

| 维度 | 亮点 |
|------|------|
| 技术广度 | 集成 6 个 AI 模型（PaddleOCR-VL、DeepSeek/LLM、IndexTTS+vLLM、Wav2Lip、MediaPipe、Whisper），覆盖视觉理解、文本生成、语音合成、唇形同步、人像分割、语音识别六大能力 |
| 技术深度 | 多模型显存协同管理（双卡 4090 运行 6 模型不 OOM）；轻量 CV 替代 GFPGAN 方案；软边缘抠图；SSE 进度单调性保证等工程细节 |
| 产品完整度 | Web 端 + 微信小程序双端；实时进度推送；管理控制台（GPU 监控、日志、定时清理）；自定义数字人（浏览器内录制） |
| 场景价值 | 面向职业院校"师资短缺、课程数字化"真实痛点，PPT 一键生成数字人讲解视频，零人工介入 |

### 1.2 当前短板

| 问题 | 影响 | 优先级 |
|------|------|--------|
| Wav2Lip 视觉效果（面部模糊、抖动） | 演示冲击力不足，评委直观感受差 | P0 |
| 生成速度慢（多页视频需数分钟） | 无法现场实时演示，影响评委体验 | P0 |
| 创新点表述不清晰 | 答辩时无法快速让评委抓住重点 | P0 |
| 缺乏竞品对比 | 无法体现项目差异化价值 | P1 |

## 二、技术优化建议

### 2.1 升级 Wav2Lip → MuseTalk（最高优先级）

| 项目 | 内容 |
|------|------|
| **当前问题** | Wav2Lip（2020 年模型）面部分辨率低、抖动感明显，在演示中是最容易被评委质疑的环节 |
| **推荐方案** | 替换为 **MuseTalk**（字节跳动开源，2024 年），唇形自然度和面部清晰度显著提升，仍属音频驱动类模型，迁移成本相对可控 |
| **备选方案** | **SadTalker** — 增加头部运动和表情驱动，视觉更生动，但计算量更大 |
| **工作量估计** | 大（2～3 周），需适配现有 video_generator.py 的接口调用方式 |
| **预期效果** | 数字人视觉质量质变，演示说服力大幅提升 |

**硬件适配评估（双卡 RTX 4090）：**

| 指标 | Wav2Lip | MuseTalk |
|------|---------|---------|
| 显存占用 | ~2～3 GB | ~8～12 GB |
| 推理速度 | ~80～120 fps | ~25～40 fps |
| 30秒视频（750帧）耗时 | ~6～10 秒 | ~20～30 秒 |
| 唇形自然度 | 中 | 高 |
| 面部清晰度 | 低（96×96 上采样） | 高（原分辨率） |
| 抖动感 | 明显 | 轻微 |
| 头部运动 | 无（静态） | 有细微自然晃动 |

**关于速度下降的实际影响：**

TTS 语音合成（约 30～60 秒/页）是整体生成流水线的瓶颈，即 TTS 耗时远大于视频合成耗时。因此将 Wav2Lip 换为 MuseTalk 虽然视频合成步骤慢约 3 倍（+20～25 秒/页），但对用户感知的总等待时间增幅有限（约 +30%）。若后续实现逐页流水线（第 N 页 TTS 完成即触发第 N 页视频生成），速度劣势可基本消除。

**迁移要点：**
- MuseTalk 接口：输入参考图像 + 驱动音频 → 输出视频帧序列
- 需适配 `wav2lip_workspce/lx/测试/services/` 下的服务封装层
- 双卡 4090（各 48GB）显存充裕，MuseTalk 与其他模型共卡无压力

**推荐落地策略：双模式并行（快速/高清）**

不必完全替换 Wav2Lip，可实现双模式切换，将工程约束转化为产品特性：

```
生成质量选项：
  ○ 快速模式（Wav2Lip）   — 日常使用，速度优先，约原有时长
  ● 高清模式（MuseTalk）  — 答辩/展示，质量优先，约慢 30%
```

- 两套推理服务分别部署在不同 GPU 上，互不干扰
- 前端在步骤配置页增加单选项，后端按参数路由到对应服务
- **本身即可作为答辩亮点**：系统支持质量/速度按需权衡

**时间不足时的备选方案：**

若集成工作量过大，可先用 MuseTalk **离线生成 2～3 个精品展示视频**用于答辩播放，主系统仍保留 Wav2Lip，答辩时说明"高清模式正在集成中"，效果已足够说服评委。

### 2.2 生成速度优化

| 优化方向 | 具体措施 |
|----------|----------|
| **流式预览** | 生成完第一页视频后立即推送可播放，不等所有页面完成 |
| **快速预览模式** | 对 ≤5 页 PPT 提供低分辨率快速生成（用于 Demo 演示） |
| **TTS 与视频并行** | 当前 TTS 全部完成后才开始视频生成，可改为逐页流水线：第 N 页 TTS 完成即触发第 N 页视频生成 |
| **目标** | 短 PPT（3～5 页）从上传到看到第一页视频 ≤ 30 秒 |

### 2.3 声音克隆 UI 完善

IndexTTS 已支持声音克隆（参考音频），但当前前端入口不够突出：
- 在数字人选择步骤中增加「克隆我的声音」入口
- 用户录制 30～60 秒参考音频，生成专属音色并保存
- 演示时用教师本人声音生成视频，冲击力极强

### 2.4 可选技术增强

| 功能 | 说明 | 工作量 |
|------|------|--------|
| 情感自适应 | 根据 PPT 内容（技术讲解/激励/介绍）自动匹配情感风格，减少用户配置负担 | 中 |
| 大屏展示端 | 新增全屏轮播路由页，适配触摸大屏，用于展厅/走廊展示 | 小 |
| 批量生成 | 支持多 PPT 排队处理，适合院校批量制作课程视频 | 中 |

## 四、答辩策略

### 4.1 创新点标准表述（3 句话版本）

> 本项目的创新体现在三个层面：
>
> **第一，流水线编排创新**——设计了异构多模型协同框架，解决了 6 个 AI 模型在双卡 GPU 环境下的显存分配、顺序调度与异常恢复问题，实现了从 PPT 到数字人视频的全自动化生成。
>
> **第二，工程优化创新**——提出了用轻量 OpenCV 算法（Unsharp Mask + 椭圆羽化融合）替代 GFPGAN 重量级面部修复模型的方案，在保持画质的前提下将生成速度提升约 10 倍，并将显存占用降低约 11GB。
>
> **第三，场景创新**——面向职业院校"师资短缺、课程数字化"的真实痛点，教师仅需上传 PPT 即可生成个人数字人讲解视频，支持声音克隆和自定义数字人形象，实现零门槛内容生产。

### 4.2 竞品对比准备

| 产品 | 优点 | 不足（对比本项目） |
|------|------|-------------------|
| HeyGen | 效果精良 | 收费，不支持私有化部署，无 PPT 自动解析 |
| D-ID | 知名度高 | 收费，无中文优化，无课程场景针对性 |
| 腾讯智影 | 国内主流 | 无声音克隆，无自定义数字人，无完整 API |
| 硅基智能 | 数字人效果好 | 收费，不开源，不可二次开发 |
| **本项目** | 开源可控、私有化部署、PPT 全流程自动化、声音克隆、自定义数字人 | 视觉效果待 MuseTalk 升级后与商业产品持平 |

### 4.3 演示脚本建议

1. **开场**（1 分钟）：一句话说清楚是什么、解决什么问题、目标用户是谁
2. **演示**（3～4 分钟）：
   - 展示上传 PPT → 配置数字人/声音 → 触发生成（不等待，切换到预生成视频）
   - 播放 2～3 个不同风格的成品视频
   - 演示控制台 GPU 监控（体现系统级工程能力）
3. **技术亮点**（2 分钟）：重点讲显存管理、流水线编排、工程优化（用架构图辅助）
4. **竞品对比**（1 分钟）：一张表说清楚差异化价值
5. **问答准备**：「模型是自己训练的吗」「创新点在哪」「实际用户反馈如何」

### 4.4 高频问题参考答案

**Q：模型是你们自己训练的吗？**
> 核心 AI 模型使用了业界成熟的开源模型，我们的创新在于**工程集成和系统设计**——如何让 6 个异构模型在有限 GPU 资源下稳定协同运行，这是真实的工程挑战，也是本项目的核心难点。

**Q：和市场上的数字人产品有什么区别？**
> 现有商业产品（HeyGen、腾讯智影等）不支持私有化部署、无法与 PPT 内容自动对齐、缺乏声音克隆能力。本项目专为职业院校场景设计，支持本地私有部署、教师声音克隆、批量课程生产，且完全开源可二次开发。

**Q：生成质量怎么样？**
> 目前视觉效果方面正在将 Wav2Lip 升级为 MuseTalk，预计提升显著。当前版本已通过 Unsharp Mask 锐化和羽化融合优化了面部清晰度，整体达到教育内容制作的实用标准。

### 5.1 必须包含的内容

- [ ] **用户测试反馈**：若有真实用户（教师/学生）使用反馈，务必纳入

### 5.2 架构图核心要素

```
PPT文件
  ↓
LibreOffice → PDF → 幻灯片图片
  ↓
PaddleOCR-VL（版式理解）
  ↓
DeepSeek LLM（口播文案生成）
  ↓
IndexTTS + vLLM（声音克隆 TTS）
  ↓
┌─────────────────────────────┐
│  并行处理                    │
│  Wav2Lip/MuseTalk（唇形同步）│
│  MediaPipe（背景抠除）        │
│  Whisper（字幕生成）          │
└─────────────────────────────┘
  ↓
FFmpeg 合成（背景+数字人+字幕+BGM）
  ↓
HLS 流式输出 → Web/小程序播放
```
