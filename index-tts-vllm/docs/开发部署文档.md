# Index-TTS-vLLM 开发部署文档

## 一、环境要求

- **Conda 环境**：`index-tts-vllm`（Python 3.12）
- **CUDA**：建议 12.4 或 12.6，与 PyTorch 版本对应
- **系统**：Linux

---

## 二、依赖安装顺序与源配置（重要）

vLLM 与 PyTorch 存在 **ABI 兼容** 问题，版本不匹配会报错：  
`ImportError: vllm/_C.abi3.so: undefined symbol: ... (c10/cuda 相关)`  
必须按以下顺序安装，并固定版本。

### 1. 创建并激活环境

```bash
conda create -n index-tts-vllm python=3.12 -y
conda activate index-tts-vllm
```

### 2. 先安装 PyTorch（与 vllm 0.16 匹配）

**必须使用 PyTorch 2.9.1 + CUDA 12.6**，否则 vllm 预编译扩展会报错。

```bash
# 清华源 + PyTorch 官方 cu126 索引
pip install "torch==2.9.1" "torchaudio==2.9.1" "torchvision==0.24.1" \
  -i https://pypi.tuna.tsinghua.edu.cn/simple \
  --extra-index-url https://download.pytorch.org/whl/cu126
```

> 若使用 cu124，可用 `--extra-index-url https://download.pytorch.org/whl/cu124`，但需与当前 vllm wheel 的 CUDA 构建一致；实测 cu126 与 vllm 0.16 兼容良好。

### 3. 安装 vllm

```bash
pip install vllm==0.16.0 -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 4. 安装项目其余依赖

```bash
cd /path/to/index-tts-vllm
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
pip install fastapi uvicorn python-dotenv soundfile  # 服务与音频保存所需
```

**不建议**在安装 vllm 之后再用 `pip install torch ...` 升级/降级 PyTorch，否则可能重新触发 ABI 不兼容。

## 三、环境变量与目录

### 1. 配置文件 `server/.env`

- **REF_AUDIO_ROOT_PATH**：参考音频根目录，**必填且路径必须存在**，否则服务启动报错。
- **MODEL_DIR**：模型 checkpoint 目录（如 `.../checkpoints/IndexTTS-2-vLLM`）。
- **DEFAULT_OUTPUT_PATH**：TTS 输出目录，与 `REF_AUDIO_ROOT_PATH` 可相同或不同。
- 所有路径建议使用**当前机器的绝对路径**（如 `/home/xxx/index-tts-vllm/outputs/`）。

### 2. 需提前创建的目录

```bash
mkdir -p outputs server/upload_temp
```

确保 `REF_AUDIO_ROOT_PATH` 和 `DEFAULT_OUTPUT_PATH` 指向的目录存在。

### 3. 配置加载方式

服务脚本会**优先从 `server/.env` 加载**环境变量（与运行时的当前目录无关），无需在项目根目录再放一份 `.env`。

## 四、启动与停止服务

### 启动（在项目根目录执行）

```bash
cd /path/to/index-tts-vllm
conda activate index-tts-vllm
# 指定 GPU，例如仅用 1 号卡
CUDA_VISIBLE_DEVICES=1 python server/tts_server_v2_batch.py
```

默认监听 `0.0.0.0:6006`，可在 `server/.env` 中修改 `HOST`、`PORT`。

### 停止与显存释放

脚本**仅停止 GPU:1 上的 TTS 服务**，不会结束 0 卡上的进程。会结束主进程及整组子进程，便于释放 1 卡显存：

```bash
./stop.sh
```

若需手动停止，需连同子进程一并结束（如先查主进程 PID，再 `kill -TERM -<主进程组>`），否则停止后显存可能仍未释放。

## 五、API 调用注意事项

### `/batch_tts_url` 的 `texts` 参数

- **类型**：表单字段，值为**合法 JSON 数组字符串**，例如：`["第一段文本", "第二段文本"]`。
- **常见错误**：
  - 使用 curl 时，若在命令行直接写长 JSON，**内部双引号**会破坏字符串，导致 400「texts参数必须是合法的JSON数组字符串」。
  - 使用 `-F "texts=@file.json"` 时，curl 会按**文件上传**发送，服务端收到的是文件对象而非字符串，导致 422。
- **推荐**：用 Python 的 `requests` 传参，或先将 JSON 写成字符串再通过 `-F "texts=<字符串>"` 传入（注意 shell 转义）。示例：

```python
import requests, json
texts = ["第一段", "第二段"]
r = requests.post(
    "http://host:6006/batch_tts_url",
    data={"texts": json.dumps(texts, ensure_ascii=False), "emo_control_method": 0, "max_text_tokens_per_sentence": 120},
    files={"spk_audio_file": open("speaker.m4a", "rb")},
    timeout=300
)
```

## 六、已知问题与项目内修复

### 1. 音频保存：torchcodec 缺失

**现象**：推理成功，但保存 wav 时报错  
`ImportError: TorchCodec is required for save_with_torchcodec ...`

**原因**：新版 `torchaudio.save()` 默认使用 torchcodec，环境未安装时会失败。

**处理**：项目已在 `indextts/infer_vllm_v2.py` 中改为使用 **soundfile** 写 wav，无需单独安装 torchcodec。确保环境中已安装：`pip install soundfile`。

### 2. 首次启动较慢

模型加载与 torch.compile 等会占用约 1～2 分钟，待日志出现 Uvicorn 监听 6006 后再调用接口。
